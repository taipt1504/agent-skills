# Summer 0.2.1 — summer-rest

Released: 2026-02-26

This release rewrites the global exception handler, replaces custom tracing with Micrometer +
OpenTelemetry, and reshapes `JsonErrorResponse`. Largest cross-cutting change in the 0.2.x
series for `summer-rest`.

## Breaking changes

### `GlobalExceptionHandler` → `SummerGlobalExceptionHandler`

The new handler implements `ErrorWebExceptionHandler` (was `WebExceptionHandler`), reads
`traceId` from MDC directly instead of `TraceIdMdcFilter.getTraceIdFromContext()`, and is
rewritten with Java 21 `switch` pattern matching.

Now handles additional exception types: `AccessDeniedException`, `AuthenticationException`,
`NoResourceFoundException`, `MethodNotAllowedException`, `NotAcceptableStatusException`,
`UnsupportedMediaTypeStatusException`, `PayloadTooLargeException`, `IllegalArgumentException`,
`IllegalStateException`. `DownstreamException` now maps to `500 INTERNAL_SERVER_ERROR` (was
`502 BAD_GATEWAY`).

### Custom tracing infrastructure removed

Deleted: `TraceIdMdcFilter`, `LoggingConfig`, `LoggingConfig.MDCUtils`, `RestLoggingConfig`,
`LogType`, `Logging`. Removed config: `f8a.common.logging.mdc.{enabled, trace-id-header,
fallback-trace-id-header}`, `mdc-enabled`, `service-name`.

Tracing now goes through Micrometer Tracing + OpenTelemetry. Services must add:

```gradle
implementation 'io.micrometer:micrometer-tracing-bridge-otel'
implementation 'io.opentelemetry:opentelemetry-exporter-otlp'
```

And configure:

```yaml
spring.reactor.context-propagation: auto
management.tracing:
  sampling.probability: 1.0
  enabled: true
management.otlp.tracing.endpoint: http://localhost:4317
```

See Summer `docs/HOW_WE_TRACE.md`.

### `JsonErrorResponse` moved to `core.exception`

Class moved from `rest-common.exception` → `core.exception`. Now includes `timestamp` (Instant)
and `details` (`List<FieldErrorDetail>`) fields plus a `FALLBACK_ERROR_STRING` constant. New
`FieldErrorDetail` record added in `core.exception` for structured field-level errors.

### `ViewableException.detail()` builder

Gains a `details` list and fluent `.detail(field, issue)` builder method. New
`CommonExceptions` entries: `NOT_ACCEPTABLE`, `UNSUPPORTED_MEDIA_TYPE`, `PAYLOAD_TOO_LARGE`.

### Refactor: `ReactiveApisixCustomizer` and `ReactiveSummerHttpSecurityCustomizer`

Both now use `SecurityErrorResponseWriter` (in `summer-jwt-resource-server`) for auth-failure
responses. `ApisixAuthenticationManager.claimType` field is now `final`. (Cross-cutting with
`summer-security`; see that skill's 0.2.1 overlay.)

### `controllerPattern` default changed

In `ControllerLoggingAspect`: from `"io.f8a.*.controller.*"` to `"execution(* *..*Controller.*(..))"`.
Custom controller packages are now caught automatically.

### `JsonUtils` deleted

The 173-line utility in `core.json` is removed. Use `ObjectMapper` directly.

## Features

- `AbstractTableValidator` base in `data-r2dbc.validator` (also referenced by `summer-data`).
- New `SecurityErrorResponseWriter` utility in `summer-jwt-resource-server`.
- New `DefaultAuthenticationException` and `DefaultJwtConverter` in `summer-jwt-resource-server`.
- Build now declares Java 21 toolchain and propagates `--enable-preview` to subprojects.

## Documentation

- New `docs/HOW_WE_TRACE.md` covering Micrometer Tracing + OpenTelemetry integration,
  dependencies, configuration, request flow, MDC keys, WebClient propagation, and local Jaeger
  setup.
- `docs/HOW_WE_AUTH.md` updated: realm `winx-local` → `ewallet-backoffice`, client-id
  `winx-user-client` → `auth-ms`, resource code `auth-ms` → `admin`.

## Migration from 0.2.0

1. Add Micrometer + OTel dependencies and the `management.tracing.*` config block.
2. Remove `f8a.common.logging.mdc.*`, `mdc-enabled`, `service-name` from `application*.yml`.
3. Update imports: `JsonErrorResponse` from `rest-common.exception` → `core.exception`.
4. Verify your `@ControllerAdvice` (if any) doesn't override the new `SummerGlobalExceptionHandler`
   exception types — defer to it where possible.
5. Replace `JsonUtils` calls with direct `ObjectMapper` usage.
