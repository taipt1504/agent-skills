# Summer 0.2.6 — summer-rest

Released: 2026-04-02

## Features

### WebFlux `String → Ufid` converter auto-registered

`SummerWebfluxConfiguration` registers a converter that lets controllers bind `Ufid` from
`@PathVariable` and `@RequestParam` automatically — accepts UUID, hex, or display-format
strings on input.

```java
@GetMapping("/orders/{id}")
Mono<OrderResponse> get(@PathVariable Ufid id) { ... }
```

URL forms accepted:
- UUID — `/orders/0195ce34-a800-cafebabe-0123456789ab`
- Hex (32 chars, deprecated in 0.2.8) — `/orders/0195CE34A800CAFEBABE0123456789AB`
- Display — `/orders/JE20260330-143025123-A7F3B2E1C9D0E1F2A3B4` (hex random portion before
  0.2.6, Crockford Base32 from 0.2.6+)

## Breaking changes

None for `summer-rest` directly. Cross-cutting Ufid-related changes (`@TXN` → `@TX`, `@Hex` →
`@Compact`, hex → Crockford Base32 in display format) live in the `summer-core` 0.2.6 overlay.

## Migration from 0.2.5

1. Drop any custom `String → Ufid` converter you registered manually.
2. (Optional) update existing endpoints to take `@PathVariable Ufid` directly instead of
   `@PathVariable String` + manual parse.
