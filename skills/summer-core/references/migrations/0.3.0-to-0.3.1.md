# Migration: Summer 0.3.0 → 0.3.1

> Outbox-heavy upgrade. Most services need YAML reshape, schema migration, and a bean cleanup.
> Per-skill detail in [`summer-data/references/versions/0.3.1.md`](../../../summer-data/references/versions/0.3.1.md).

## What's changing

- **Outbox config schema** reshape: `cdc.*` (top-level) → `publisher.cdc.*`; scheduler tuning
  moves to `publisher.scheduler.*`.
- **Debezium storage moves JDBC → Kafka** (BREAKING). Dependency swap + replication slot reset
  required.
- **`KafkaOutboxPublisher` auto-wired** when `KafkaTemplate` is on the classpath. You can now
  delete your custom publisher.
- **Outbox publish header `ob.lsn` is binary** (BREAKING). Consumers binding `@Header Long lsn`
  must recompile against 0.3.1.
- **`OutboxEventPublisher.publish(event, headers)` takes `Map<String, byte[]>`** (BREAKING for
  custom publishers).
- **`outbox_events.lsn` column added** — preserves Debezium WAL LSN across retries.
- **New `summer-kafka-consumer` module** — LSN-watermark idempotency. Optional adoption.
- **Class renames** — `DebeziumCdcEventHandler` → `DebeziumCdcEventDispatcher`, etc.

## Step 1 — schema migration

```sql
ALTER TABLE outbox_events ADD COLUMN IF NOT EXISTS lsn BIGINT;
```

Pre-upgrade rows get `NULL`; retries fall back to old epoch-nanos behaviour (safe — matches
pre-0.3.1 semantics).

## Step 2 — YAML reshape

Apply this mapping table (sample-driven examples in
[`summer-data/references/versions/0.3.1.md`](../../../summer-data/references/versions/0.3.1.md)):

| Before (0.3.0) | After (0.3.1) |
|---|---|
| `f8a.outbox.publisher.cron` | `f8a.outbox.publisher.scheduler.cron` |
| `f8a.outbox.publisher.batch-size` | `f8a.outbox.publisher.scheduler.batch-size` |
| `f8a.outbox.publisher.initial-delay` | **removed** (was dead config) |
| `f8a.outbox.cdc.*` (top-level) | `f8a.outbox.publisher.cdc.*` |
| `f8a.outbox.publisher.kafka.topic-prefix` | `f8a.outbox.publisher.topic-prefix` (flattened) |
| `f8a.outbox.cdc.publish-timeout-seconds: 30` | `f8a.outbox.publisher.cdc.publish-timeout: 30s` |
| `f8a.outbox.cdc.database-server-name` | `f8a.outbox.publisher.cdc.topic-prefix` |

## Step 3 — Debezium storage swap (CDC users only)

If `f8a.outbox.publisher.mode: cdc`:

1. **Drop the existing replication slot.** Debezium re-creates it at the current LSN on first
   boot:
   ```sql
   SELECT pg_drop_replication_slot('outbox_slot');
   ```
2. **Create two compacted Kafka topics** (replication factor matching cluster policy):
   ```bash
   kafka-topics --create --topic debezium.outbox.offsets \
     --partitions 1 --replication-factor 3 --config cleanup.policy=compact
   kafka-topics --create --topic debezium.schema.history \
     --partitions 1 --replication-factor 3 --config cleanup.policy=compact
   ```
3. **Swap dependency**:
   ```diff
   - implementation 'io.debezium:debezium-storage-jdbc'
   + implementation 'io.debezium:debezium-storage-kafka'
   ```
4. Add to YAML:
   ```yaml
   f8a.outbox.publisher.cdc:
     bootstrap-servers: ${KAFKA_BOOTSTRAP}    # defaults to spring.kafka.bootstrap-servers
     offset-storage-topic: debezium.outbox.offsets
     offset-storage-replication-factor: 3
     schema-history-topic: debezium.schema.history
   ```

## Step 4 — drop custom Kafka publisher (if applicable)

`KafkaOutboxPublisher` ships as the default. If you had a hand-rolled
`@Bean OutboxEventPublisher` that just produced to Kafka, delete it. Auto-config detects
`KafkaTemplate` + `f8a.outbox.publisher.queue=kafka` (default) and registers the built-in.

To keep your custom publisher: it still wins because of `@ConditionalOnMissingBean`. Just
ensure the signature uses `Map<String, byte[]>` for headers (BREAKING — see step 5).

## Step 5 — fix custom `OutboxEventPublisher` signatures

```java
// Before
Mono<Void> publish(OutboxEvent event, Map<String, String> headers);

// After
Mono<Void> publish(OutboxEvent event, Map<String, byte[]> headers);
```

Drop any `headerValue.getBytes(UTF_8)` — values are already bytes.

## Step 6 — verify consumer header binding

Consumers binding `@Header Long lsn` must recompile against 0.3.1. The pre-release decimal
string format silently produced wrong values; the new 8-byte big-endian format is decoded
natively by Spring Messaging's `byte[] → Long` converter.

```java
@KafkaListener(topics = "orders.created")
void onOrderCreated(OrderCreatedEvent event, @Header(name = "ob.lsn") Long lsn) { ... }
```

## Step 7 (optional) — adopt LSN-watermark idempotency

```diff
+ implementation 'io.f8a.summer:summer-kafka-consumer'
+ implementation 'io.f8a.summer:summer-kafka-consumer-autoconfigure'
```

Schema:
```sql
-- See HOW_WE_IDEMPOTENCY_ON_CONSUMER.md in the Summer docs for the full DDL.
CREATE TABLE outbox_consumer_watermark (
    consumer_group TEXT NOT NULL,
    topic TEXT NOT NULL,
    partition INT NOT NULL,
    last_lsn BIGINT NOT NULL,
    updated_at TIMESTAMPTZ NOT NULL DEFAULT NOW(),
    PRIMARY KEY (consumer_group, topic, partition)
);
```

Wrap business logic with `isProcessed → process → recordProcessed` in one reactive
transaction.
