# Summer 0.2.5 — summer-core (UFID introduced)

Released: 2026-03-30

## Features

### Unified Financial ID (UFID)

128-bit, chronologically sortable, collision-safe identifier for financial entities. Bit
layout: 48-bit millisecond timestamp in the high bits + 80-bit cryptographic random in the low
bits. No coordination needed across nodes.

Core classes in `summer-core` (`io.f8a.summer.core.domain.types.ufid`):

- `Ufid` — immutable value type. Methods (0.2.5 surface):
  - `generate()`, `fromBytes()`, `fromHex()`, `fromUUID()`
  - `toBytes()`, `toHex()`, `toUUID()`
  - `getTimestamp()`, `getInstant()`
  - Implements `Comparable<Ufid>` for chronological sorting.
  - Default Jackson serialization uses UUID string.
- `UfidFormat` — display format utility.
  - `format(ufid, "JE")` → `JE20260330-143025123-A7F3B2E1C9D0E1F2A3B4` (20 hex random; replaced
    by 26-char Crockford Base32 in 0.2.6).
  - `parse(display)` round-trips back to the original `Ufid`.

### `summer-payment-sdk` module added

UFID Jackson serialization annotations and shared payment domain enums.

Predefined prefix annotations apply display format with entity-specific prefixes:
- `@JE` — Journal Entry (`JE20260330-...`)
- `@TXN` — Transaction (`TXN20260330-...`) — renamed `@TX` in 0.2.6
- `@SE` — Settlement (`SE20260330-...`)
- `@UfidPrefix("XX")` — custom prefix; usable as a meta-annotation to define new aliases.
- `@Hex` — compact 32-character uppercase hex (`0195CE34A800CAFEBABE0123456789AB`) — renamed
  `@Compact` and reformatted to 26-char Crockford Base32 in 0.2.6.

Without annotation, `Ufid` serialises as a standard UUID string.

### Shared payment domain enums

`io.f8a.summer.payment.domain` introduces a two-layer transaction taxonomy following MoMo-style
product-driven classification:

| Enum | Values |
|---|---|
| `TransactionType` | TOPUP, WITHDRAWAL, P2P_TRANSFER, BANK_TRANSFER, QR_PAYMENT, BILL_PAYMENT, MERCHANT_PAYMENT, REFUND, FEE, SETTLEMENT, INCENTIVE |
| `LedgerOperation` | TRANSFER, HOLD, CAPTURE, RELEASE, REVERSAL, ADJUSTMENT (CAPTURE renamed POST in 0.2.6) |
| `LedgerEntryDirection` | DEBIT, CREDIT |
| `TransactionStatus` | INITIATED, PENDING, PROCESSING, SUCCEEDED, FAILED, CANCELLED |
| `TransactionChannel` | IN_WALLET, BANK_PARTNER |

### `UfidConverter` for R2DBC

In `summer-data-r2dbc`. R2DBC `@ReadingConverter` (UUID → Ufid) and `@WritingConverter` (Ufid →
UUID), auto-registered in `SummerR2dbcAutoConfiguration`. Store `Ufid` fields in PostgreSQL
UUID columns with zero configuration.

See Summer `docs/HOW_WE_UFID.md`.

## Breaking changes

None — this is a brand-new module + types.

## Migration from 0.2.4

Add the dependency only if you need it:

```gradle
implementation 'io.f8a.summer:summer-payment-sdk'   // for @JE, @TXN, @Compact, @Hex etc.
implementation 'io.f8a.summer:summer-data-r2dbc'    // already pulled in by summer-data-autoconfigure
```

Adopt `Ufid` for new financial entity IDs. Existing `UUID`-typed columns are reusable —
`UfidConverter` round-trips both ways.
