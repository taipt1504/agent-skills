# Summer 0.2.1 — summer-core

Released: 2026-02-26

## Breaking changes

### `JsonErrorResponse` moved package

| Before | After |
|---|---|
| `io.f8a.summer.rest.common.exception.JsonErrorResponse` | `io.f8a.summer.core.exception.JsonErrorResponse` |

Now includes new fields:
- `timestamp` (`Instant`)
- `details` (`List<FieldErrorDetail>`)
- `FALLBACK_ERROR_STRING` constant

### `JsonUtils` deleted

The 173-line utility in `core.json` is removed. Use `ObjectMapper` directly. `BusinessChange`
model in `core.audit.model` and `ContextConstants.TRACE_ID_KEY` are also deleted.

### `ContextConstants.TRACE_ID_KEY` deleted

Read `traceId` from MDC directly (Micrometer Tracing populates it).

## Features

### `ViewableException.detail()` builder

```java
throw new ViewableException("validation.error", BAD_REQUEST)
    .detail("amount", "must be positive")
    .detail("currency", "unknown ISO code");
```

The `details` list flows into `JsonErrorResponse.details` via `SummerGlobalExceptionHandler`.

### `FieldErrorDetail` record

New record in `core.exception` for structured field-level errors:

```java
public record FieldErrorDetail(String field, String issue) {}
```

### `CommonExceptions` additions

New entries: `NOT_ACCEPTABLE` (406), `UNSUPPORTED_MEDIA_TYPE` (415), `PAYLOAD_TOO_LARGE` (413).

### Build: Java 21 toolchain

`build.gradle` now declares the Java 21 toolchain and propagates `--enable-preview` to all
subprojects. Google Artifact Registry repo registration is now guarded behind null-checks on
config properties (so local builds without GAR creds don't fail).

## Migration from 0.2.0

1. Update imports: `import io.f8a.summer.core.exception.JsonErrorResponse;`
2. Replace `JsonUtils.x(...)` calls with direct `ObjectMapper` calls.
3. Delete any usage of `BusinessChange` or `ContextConstants.TRACE_ID_KEY`.
4. (Optional) start using `.detail(field, issue)` on thrown `ViewableException` for richer
   error responses.
