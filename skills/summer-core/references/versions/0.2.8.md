# Summer 0.2.8 — summer-core (UFID encoding cleanup)

Released: 2026-04-12

Hex encoding is dropped in favour of Crockford Base32 across all UFID surfaces. Class renamed
`UfidFormat` → `UfidDisplay`. Many UFID utility additions land in `summer-payment-sdk`.

## Breaking changes

### `UfidFormat` → `UfidDisplay`

All static methods unchanged, only the class name changed.

```java
// Before
import io.f8a.summer.core.domain.types.ufid.UfidFormat;

// After
import io.f8a.summer.core.domain.types.ufid.UfidDisplay;
```

### `@Compact` outputs 26-char Crockford Base32 (was 32-char hex)

```
Before: "0195CE34A800CAFEBABE0123456789AB"  (32 chars)
After:  "04NN8D500PFNAXM04HB4F5A18"          (26 chars)
```

If consumers persisted `@Compact`-encoded strings, the format changes — re-encode or store the
UUID form going forward.

### `Ufid.fromHex()` and `Ufid.toHex()` removed

Use `Ufid.fromBase32()`, `Ufid.fromUUID()`, or `Ufid.fromString()` (which auto-detects format).
`Ufid.toString()` now returns the UUID string representation (matches `@JsonValue`
serialisation and DB storage).

### `Ufid.fromString()` no longer detects 32-char hex

Supported formats: UUID (36 chars), Base32 (26 chars), and display format (with or without
prefix). Hex inputs from older systems must be converted to UUID first.

### Removed `f8a.outbox.publisher.enabled`

(Cross-cutting with `summer-data` 0.2.8 overlay — full migration there.)

## Features

### `Ufid.toBase32()` / `Ufid.fromBase32()`

Encode/decode the full 128-bit UFID as a 26-character Crockford Base32 string. Case-insensitive
with Crockford typo tolerance (O→0, I/L→1).

### `Ufid.fromUInt128(BigInteger)` / `Ufid.toUInt128()`

Convert between UFID and unsigned 128-bit integer. Useful for TigerBeetle interop.

### `UfidUInt128Deserializer` (in `summer-payment-sdk`)

Jackson deserialiser that parses a numeric uint128 string into a `Ufid`. Zero value
deserialises to `null`.

### Default `@JsonCreator` accepts UUID/Base32/display

Plain `Ufid` fields in request bodies accept UUID, Base32, or display format on input without
needing a Jackson annotation.

### `UfidDeserializer` honours `@UfidPrefix` strictly

Now implements `ContextualDeserializer` — when a `@UfidPrefix` annotation is present, the
deserialiser validates the incoming string starts with the expected prefix and strips it
directly instead of scanning for the first digit.

### `LedgerAccountType` Javadoc

Comprehensive Javadoc with user/system account classification tables, ledger codes, and GL
mapping documentation.

## Migration from 0.2.6

1. Search-replace `UfidFormat` → `UfidDisplay`.
2. Search for `Ufid.fromHex(` and `Ufid.toHex(` — replace with `fromString` / UUID round-trip.
3. Audit persisted strings in `@Compact`-annotated columns — they'll mismatch after upgrade if
   consumers re-encode. Prefer storing UUID, format on output only.
4. (Optional) drop now-unnecessary `@JsonDeserialize` annotations on `Ufid` request fields —
   the default accepts all common formats.

## Verification

- `Ufid.toBase32()` returns a 26-char Crockford string with no `I`, `L`, `O`, `U`.
- Old hex inputs throw on parse — confirm migrations handled all consumer-facing formats.
- `@UfidPrefix` mismatches surface as Jackson errors rather than mis-interpreted IDs.
