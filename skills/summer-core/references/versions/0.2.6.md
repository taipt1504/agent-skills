# Summer 0.2.6 — summer-core (UFID Crockford Base32)

Released: 2026-04-02

This release switches UFID's display random portion from 20-char hex to 16-char Crockford
Base32 and renames the Jackson annotations to fit the new format. Mostly affects
`summer-payment-sdk` consumers but also touches `summer-core` enums.

## Breaking changes

### `@TXN` → `@TX`, `@Hex` → `@Compact` (annotation renames)

```java
// Before
@TXN Ufid id;
@Hex Ufid id;

// After
@TX Ufid id;
@Compact Ufid id;
```

Update imports + field annotations across DTOs and entities.

### UFID display format: Crockford Base32 (was hex)

Random portion is now 16 Base32 characters (was 20 hex characters).

```
{PREFIX}{YYYYMMDD}-{HHmmssSSS}-{BASE32_16}
JE20260402-143025123-A7F3B2E1C9D0E1F2
```

Crockford's alphabet (`0-9A-Z` minus `I, L, O, U`) avoids ambiguous characters; decoding is
case-insensitive with typo tolerance (`O→0`, `I/L→1`).

### `UfidFormat.parse()` → `UfidFormat.parseDisplayFormat()`

```java
// Before
UfidFormat.parse(displayString);

// After
UfidFormat.parseDisplayFormat(displayString);
```

(In 0.2.8 the class itself is renamed to `UfidDisplay`.)

### `LedgerOperation.CAPTURE` → `LedgerOperation.POST`

Rename in `summer-payment-sdk` enum. Update enum constant references.

### `PhoneNumber.toString()` masks PII

Now returns a PII-safe masked representation (e.g. `+84 91****678`). Use `getValue()` for the
full unmasked number. This affects logging and debug output. JSON serialization via
`@JsonValue` is unchanged.

## Features

### `Ufid.fromString(String)` — universal parser

Auto-detects format: UUID, hex (32 chars), or display format with/without prefix. Accepts any
format that UFID can produce.

```java
Ufid.fromString("0195ce34-a800-...")        // UUID
Ufid.fromString("0195CE34A800CAFE...")      // hex (still accepted in 0.2.6, removed 0.2.8)
Ufid.fromString("JE20260402-143025-A7F3...")  // display
Ufid.fromString("20260402-143025-A7F3...")    // display, prefix optional
```

### WebFlux `String → Ufid` converter (auto-registered)

In `SummerWebfluxConfiguration`. Enables `@PathVariable Ufid id` out of the box. Cross-cutting
with `summer-rest`.

### `UfidUInt128Serializer` and `@UInt128` (in `summer-payment-sdk`)

Serialises `Ufid` as a decimal uint128 string for systems that represent 128-bit IDs as
integers (e.g. TigerBeetle CDC events).

## Migration from 0.2.5

1. Search-replace `@TXN` → `@TX` and `@Hex` → `@Compact` across the codebase.
2. Update `UfidFormat.parse(...)` calls → `UfidFormat.parseDisplayFormat(...)`.
3. Replace `LedgerOperation.CAPTURE` → `LedgerOperation.POST`.
4. Audit any logs that printed `PhoneNumber` directly — they now mask the middle digits. If
   you genuinely need the unmasked form (for example writing to a controlled audit channel),
   call `getValue()` explicitly.
5. Persisted display-format strings from before 0.2.6 (20-char hex random) won't round-trip
   through `parseDisplayFormat()` anymore. If you stored display strings (rare — usually you
   store the UUID), re-encode via the new format.

## Verification

- Compile-time errors flush out any `@TXN`/`@Hex` usages.
- Display strings produced after upgrade are 26 + prefix chars total (was 30+).
- `PhoneNumber#toString` output starts redacting in logs.
