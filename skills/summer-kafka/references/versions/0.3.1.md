# Summer 0.3.1 — summer-kafka (module introduction)

Released: 2026-04-20

`summer-kafka-consumer` + `summer-kafka-consumer-autoconfigure` ship in 0.3.1. This is the introduction overlay — there's no prior version to migrate from. Cross-cutting outbox changes (binary `ob.lsn` header, `Map<String, byte[]>` headers, config reshape) live in `summer-data/references/versions/0.3.1.md`.

## Features

### LSN-watermark consumer idempotency

`OutboxConsumerIdempotency` + `R2dbcOutboxConsumerIdempotency` — one row per `(consumer_group, topic, partition)` in `outbox_consumer_watermark`. Scales with topic-partition count, not message volume. Replaces per-event dedup tables and the `DuplicateKeyException`-on-domain-key pattern.

Watermark advances only forward. Replays at the same or lower LSN return `isProcessed=true` and the listener skips the business call.

### `DefaultErrorHandler` with exponential-backoff retry + DLT

Auto-wired error handler matching `ExponentialBackOffWithMaxRetries` semantics. Failed records retry per `f8a.kafka.consumer.retry.*`, then go to `<topic>.DLT` via `DeadLetterPublishingRecoverer`.

### `DefaultErrorHandlerCustomizer`

`@FunctionalInterface` — register a bean to add non-retryable exceptions (validation errors, deserialization errors) without rewriting the handler.

### `dltKafkaTemplate` bean

Dedicated `KafkaTemplate` for the DLT recoverer with `enable.idempotence=false, acks=1`. DLT writes are fire-and-forget — no value in paying the idempotent producer's latency tax. Override via `@Bean("dltKafkaTemplate")`.

### `SummerKafkaConsumerFactories` static helpers

- `jsonConsumerFactory(KafkaProperties, Class<V> valueType)` — typed consumer factory.
- `jsonListenerContainerFactory(KafkaProperties, ConsumerFactory, DefaultErrorHandler)` — paired container factory.

### Cross-cutting: `ob.lsn` header is 8-byte big-endian binary

Producers in 0.3.1+ emit `ob.lsn` as binary (was UTF-8 decimal pre-release). Consumers binding `@Header("ob.lsn") long` work correctly only against 0.3.1+ producers — older producers silently produce wrong values.

## Breaking changes

None (module introduction).

## Config schema

```yaml
f8a:
  kafka:
    consumer:
      idempotency:
        enabled: true                # default
        validate-on-startup: true    # default
      retry:
        max-attempts: 3
        initial-interval: 1s
        multiplier: 2.0
        max-interval: 30s
```

No `enabled` flag at the module level — presence of `summer-kafka-consumer-autoconfigure` on the classpath is the opt-in.

## Migration from 0.3.0 (adopting the module)

1. Add the dependencies:
   ```gradle
   implementation 'io.f8a.summer:summer-kafka-consumer'
   implementation 'io.f8a.summer:summer-kafka-consumer-autoconfigure'
   ```
2. Apply the schema (column names are validated literally by `ConsumerWatermarkValidator`):
   ```sql
   CREATE TABLE outbox_consumer_watermark (
       consumer_group  VARCHAR(255) NOT NULL,
       topic           VARCHAR(255) NOT NULL,
       partition       INTEGER      NOT NULL,
       last_lsn        BIGINT       NOT NULL,
       last_event_id   UUID,
       last_updated_at TIMESTAMPTZ  NOT NULL DEFAULT now(),
       PRIMARY KEY (consumer_group, topic, partition)
   );
   ```
3. Replace your existing dedup table calls with `OutboxConsumerIdempotency.isProcessed` / `recordProcessed`, both run inside a `TransactionalOperator.transactional(...)` boundary together with the business write.
4. Remove any custom `DefaultErrorHandler` wiring that just adds backoff + DLT — the autoconfigure provides it. Keep yours only if you have non-default retry semantics; otherwise register a `DefaultErrorHandlerCustomizer` bean for exception-policy tweaks.
5. Confirm producers are on `summer-data-outbox` 0.3.1+ so `ob.lsn` is binary.

## Verification

- `outbox_consumer_watermark` row appears for each new `(group, topic, partition)` the consumer touches.
- A deliberate re-delivery (e.g. consumer restart before commit) is skipped — log line `Skipping already-processed record lsn=<n>` (or equivalent — depends on your call site).
- A poison message reaches `<topic>.DLT` after `max-attempts` retries with `RetryableTopic`-style delays.
- `ConsumerWatermarkValidator` logs `Schema OK` at startup, or fails fast with a clear column-list mismatch when the table is misshapen.
