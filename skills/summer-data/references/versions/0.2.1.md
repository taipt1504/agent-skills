# Summer 0.2.1 — summer-data

Released: 2026-02-26

## Breaking changes

### `auditNonEntity` parameter order swap

```java
// Before
auditService.auditNonEntity(intent, action, comment);

// After
auditService.auditNonEntity(action, intent, comment);
```

### Removed config properties (schema is now auto-detected)

| Removed | Replacement |
|---|---|
| `f8a.outbox.validate-schema` | Auto via `SELECT current_schema()` |
| `f8a.audit.validate-schema` | Auto via `SELECT current_schema()` |

### Removed types

- `BusinessChange` model (`core.audit.model`).
- Embedded Flyway scripts in `data-audit` (`V1000__audit_logs_schema.sql`, `V1001__audit_log_add_action.sql`).
  Services must include `audit_log` DDL in their own migrations going forward.

## Features

### `audit(AuditLog)` builder

Build a fully custom `AuditLog`; the service auto-fills any null fields (id, timestamps, actor
info from security context, request info from reactor context).

```java
auditService.audit(AuditLog.builder()
    .action("LOGIN")
    .intent("USER_REQUEST")
    .build());
```

`auditCustom(...)` is now deprecated in favour of the builder.

### `AbstractTableValidator` base class

Lives in `data-r2dbc.validator`. Auto-detects schema via `SELECT current_schema()`, validates
table existence + column names + column types against an expected map, throws
`IllegalStateException` with the migration script on failure.

### `AuditTableValidator` (new)

Validates the `audit_log` table (23 columns) on startup. Enabled by default via
`f8a.audit.validate-on-startup=true` (opt-out with `false`).

### `OutboxTableValidator` enhanced

Now extends `AbstractTableValidator` and validates 9 columns (was just existence). Enabled by
default via `f8a.outbox.validate-on-startup=true`.

## Bug fixes

- **`AuditAspect` Flux handling.** Previously audited N times (once per element via `flatMap`).
  Now audits exactly once after the Flux completes via `concatWith`.
- **`AuditAspect` Mono handling.** Previously double-subscribed (`switchIfEmpty` + `flatMap`
  both triggered audit). Now uses `delayUntil` + `switchIfEmpty` for a single audit call.
- Add warning log when `@Audit` is applied to non-reactive methods. Previously failed silently.

## Migration from 0.2.0

1. Search-replace any direct `auditNonEntity(intent, action, ...)` calls to swap the first two args.
2. Drop `f8a.outbox.validate-schema` and `f8a.audit.validate-schema` from `application*.yml`.
3. Add `audit_log` DDL to your own Flyway migrations (Summer no longer ships them).
4. Migrate `auditCustom(...)` calls to `audit(AuditLog.builder()...)`. Both still work, but the
   builder is the canon.
