# Summer 0.3.1 — summer-data

Released: 2026-04-20

This release introduces the `summer-kafka-consumer` module, ships `KafkaOutboxPublisher` as
auto-wired default, and reshapes outbox config one more time. Debezium storage moves JDBC →
Kafka.

## Features

### `summer-kafka-consumer` module — LSN-watermark idempotency

Replaces per-event dedup tables and the domain-unique-key + `DuplicateKeyException` pattern
with a single row per `(consumer_group, topic, partition)` — matching Kafka's native
offset-tracking granularity. Scales with topic-partition count, not message volume.

Schema lives in `outbox_consumer_watermark` (see `docs/HOW_WE_IDEMPOTENCY_ON_CONSUMER.md` in
Summer docs). Inject `OutboxConsumerIdempotency` + `TransactionalOperator` into your
`@KafkaListener` and wrap the business call with `isProcessed → process → recordProcessed` in
one reactive transaction.

### Built-in `KafkaOutboxPublisher`

Ships in `summer-data-outbox`. Auto-wires an `OutboxEventPublisher` bean when `KafkaTemplate` is
on the classpath and `f8a.outbox.publisher.queue=kafka` (default). Removes ~50 lines of
copy-pasted boilerplate per service. Override by declaring your own `@Bean OutboxEventPublisher`
(autoconfig backs off) or set `queue` to any non-`kafka` value to disable.

### Built-in Kafka consumer defaults (`summer-kafka-consumer-autoconfigure`)

- `DefaultErrorHandler` with exponential-backoff retry + DLT publishing
  (`f8a.kafka.consumer.retry.*`).
- `DefaultErrorHandlerCustomizer` functional interface — add non-retryable exceptions without
  replacing the whole handler.
- `dltKafkaTemplate` bean with `enable.idempotence=false, acks=1` (DLT is fire-and-forget — no
  point paying the idempotent-producer latency tax). Overridable via `@Bean("dltKafkaTemplate")`.
- DLT destination resolver routes to `<source-topic>.dlt` (lowercase).
- `SummerKafkaConsumerFactories` static helpers — `jsonConsumerFactory(props, valueType)` +
  `jsonListenerContainerFactory(props, cf, errorHandler)`.

### Fail-loud outbox autoconfig

Removed silent `@ConditionalOnBean(OutboxEventPublisher.class)` guards from dispatchers, retry
task, and CDC config. When `f8a.outbox.enabled=true` and no publisher is available, Spring fails
at startup with a clear error instead of silently skipping bean creation.

### CDC `lsn` preserved across retries

New `outbox_events.lsn BIGINT` column persists the Debezium `source.lsn` on CDC publish
failure. `OutboxRetryTask` re-emits the same `ob.lsn` Kafka header on retry so consumer
watermarks stay consistent (previously, retries emitted `createdAt.toEpochNanos()` which dwarfed
real WAL LSNs and silently filtered subsequent CDC events).

```sql
ALTER TABLE outbox_events ADD COLUMN IF NOT EXISTS lsn BIGINT;
```

## Breaking changes

### Kafka header `ob.lsn` is now 8-byte big-endian binary

Was UTF-8 decimal string in pre-release 0.3.1 previews. Matches Spring Messaging's default
`byte[] → Long` converter, so consumers bind `@Header Long lsn` with zero config. The prior
decimal-string format silently produced the **wrong** numeric value under `@Header Long`.

### `OutboxEventPublisher.publish(event, headers)` takes `Map<String, byte[]>`

Was `Map<String, String>`. Custom publishers: change the parameter type, drop any
`.getBytes(UTF_8)` calls — values are already bytes.

### Outbox config reshape

Mode-specific tuning moved into nested blocks; CDC config promoted into `publisher.*`.

| Before (0.2.8 → 0.3.0) | After (0.3.1) |
|---|---|
| `f8a.outbox.publisher.cron` | `f8a.outbox.publisher.scheduler.cron` |
| `f8a.outbox.publisher.batch-size` | `f8a.outbox.publisher.scheduler.batch-size` |
| `f8a.outbox.publisher.initial-delay` | **removed** (was dead config) |
| `f8a.outbox.cdc.*` (top-level) | `f8a.outbox.publisher.cdc.*` |
| `f8a.outbox.publisher.kafka.topic-prefix` | `f8a.outbox.publisher.topic-prefix` (flattened) |
| `f8a.outbox.cdc.publish-timeout-seconds: 30` | `f8a.outbox.publisher.cdc.publish-timeout: 30s` (Duration) |
| `f8a.outbox.cdc.database-server-name` | `f8a.outbox.publisher.cdc.topic-prefix` |

### Debezium storage moved JDBC → Kafka

Offset + schema-history live on the Kafka cluster instead of the app DB.

```yaml
f8a.outbox.publisher.cdc:
  bootstrap-servers: ${KAFKA_BOOTSTRAP}      # defaults to spring.kafka.bootstrap-servers
  offset-storage-topic: debezium.outbox.offsets
  offset-storage-replication-factor: 3       # raise for production
  schema-history-topic: debezium.schema.history
```

**Migration:** create the two compacted topics on your broker and **drop the replication slot
before first boot** (Debezium re-creates it at current LSN). Dependency
`debezium-storage-jdbc` replaced by `debezium-storage-kafka`.

### New publisher axis: `f8a.outbox.publisher.queue`

Default `kafka`. Universal transport selector — applies in both scheduler and CDC modes. Set to
any non-`kafka` value to disable the built-in `KafkaOutboxPublisher` and bring your own.

### Class renames / moves

Only matters for direct references:

| Before (0.2.8 → 0.3.0) | After (0.3.1) |
|---|---|
| `DebeziumCdcEventHandler` | `DebeziumCdcEventDispatcher` |
| `OutboxPublisherTask` | `OutboxSchedulerDispatcher` |
| bean `outboxPublisherTask` | `outboxSchedulerDispatcher` |
| `SchedulerHeaderBuilder` (scheduler/, pkg-private) | `OutboxHeaderBuilder` (publisher/, public) |
| `DebeziumCdcProperties` | `OutboxProperties.Cdc` (nested) |

## Additive config

- New `f8a.kafka.consumer.{idempotency.{enabled, validate-on-startup}, retry.{max-attempts, initial-interval, multiplier, max-interval, dlt-suffix}}`.
- New `f8a.outbox.publisher.{queue, topic-prefix, scheduler.*, cdc.*}` structure (see breaking-changes table).

## Migration from 0.3.0

1. **Schema:** apply `ALTER TABLE outbox_events ADD COLUMN IF NOT EXISTS lsn BIGINT;`.
2. **YAML reshape** per the table above. The biggest churn is the `cdc.*` top-level → nested
   under `publisher.*`.
3. **Custom `OutboxEventPublisher`**: change header map type to `Map<String, byte[]>` and drop
   any UTF-8 encoding.
4. **CDC users**: drop the existing replication slot, create the new Kafka storage topics, swap
   the `debezium-storage-jdbc` dep for `debezium-storage-kafka`.
5. **Consumers binding `@Header Long lsn`**: recompile against 0.3.1 — the binary format gives
   correct values automatically.
6. (Optional) add `summer-kafka-consumer` for LSN-watermark idempotency.
7. (Optional) drop your custom `KafkaOutboxPublisher` bean — the auto-wired one usually
   suffices.

## Verification

- Outbox publishes succeed without a custom `OutboxEventPublisher` bean (auto `KafkaOutboxPublisher`).
- `ob.lsn` header is 8 bytes (binary), not decimal string.
- Replication slot retains LSN through a CDC outage; failed events resume from the same `lsn`
  on retry.
- `outbox_consumer_watermark` table populated on first message processed.
