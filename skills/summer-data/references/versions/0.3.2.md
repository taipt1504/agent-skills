# Summer 0.3.2 — summer-data

Released: 2026-04-20

Small, additive release — no breaking changes for outbox/audit/kafka-consumer.

## Features

### Debezium Kafka storage inherits SASL/SSL from `spring.kafka.*`

When `publisher.mode=cdc`, Debezium's internal offset-storage and schema-history Kafka clients
now pick up the full Kafka client config resolved by Spring's `KafkaProperties` — bootstrap
servers, `security.protocol`, `sasl.mechanism`, `sasl.jaas.config`, SSL truststore/keystore,
etc. Propagated to both the top-level engine config (for `KafkaOffsetBackingStore`) and
mirrored under `schema.history.internal.kafka.*` (for `KafkaSchemaHistory`, which only reads
that prefix).

Without this, SASL-protected clusters rejected Debezium's internal clients at handshake with
`SaslAuthenticationException`. Services on a secured cluster: just configure
`spring.kafka.*` once — Debezium clients now follow.

### Swagger BOM alignment in `summer-platform`

Added explicit constraints for `swagger-annotations`, `swagger-core-jakarta`, and
`swagger-models-jakarta` pinned to `io.swagger.core.v3:2.2.35`. Prevents runtime
`NoSuchMethodError` on `Schema.dependentSchemas()` when a transitive dependency (Keycloak admin
client, legacy Springfox leftovers) drags in an older `swagger-annotations` jar that lacks
methods springdoc's newer `swagger-core` expects.

## Breaking changes

### Dropped `f8a.outbox.publisher.cdc.bootstrap-servers`

CDC storage always lives on the cluster defined by `spring.kafka.bootstrap-servers` — there is
no cross-cluster override path. If you were pointing CDC at a different cluster, restructure to
use the same Spring Kafka cluster.

### Dropped `f8a.outbox.publisher.cdc.additional-properties`

Debezium engine config is no longer user-extensible via this map; all Kafka client settings
flow through `spring.kafka.*`. Reduces config surface to one source of truth.

### Startup fails when CDC has no Kafka bootstrap-servers

Startup fails with a clear message if `spring.kafka.bootstrap-servers` is missing when
`publisher.mode=cdc`. Previously misconfiguration was silent.

### Direct API change

`DebeziumCdcEngine` constructor now takes `Map<String, Object> kafkaClientConfig` instead of
`String bootstrapServers` (only matters for direct references — autoconfig is unchanged from
the consumer perspective).

## Migration from 0.3.1

1. Confirm `spring.kafka.bootstrap-servers` is set when `f8a.outbox.publisher.mode=cdc` (now
   required).
2. Remove `f8a.outbox.publisher.cdc.bootstrap-servers` — it's dropped silently if present, but
   keeping it is dead config.
3. Remove `f8a.outbox.publisher.cdc.additional-properties` — supply Kafka client settings via
   `spring.kafka.*` instead.
4. Direct callers of `new DebeziumCdcEngine(...)` (rare): pass the resolved
   `Map<String, Object>` from `KafkaProperties.buildProducerProperties()` (or similar).

## Verification

- App starts on a SASL-protected Kafka cluster without manual JAAS config in
  `f8a.outbox.publisher.cdc.*`.
- Debezium offsets land on the configured Kafka cluster, not the app DB.
- Swagger UI loads without `NoSuchMethodError` on `Schema.dependentSchemas()`.
