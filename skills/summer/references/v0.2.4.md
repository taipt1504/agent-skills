# Summer Framework — v0.2.4

Released: 2026-03-22

## Breaking Changes from 0.2.3

### Keycloak config moved to shared block

Keycloak connection properties moved from `sync-role` to a shared `keycloak` block:

```yaml
# Before (0.2.3)
f8a.security.apisix.resource-server:
  sync-role:
    server-url: https://kc.example.com
    realm: master
    client-id: admin-cli
    client-secret: secret

# After (0.2.4)
f8a.security.apisix.resource-server:
  keycloak:
    server-url: https://kc.example.com
    realm: master
    client-id: admin-cli
    client-secret: secret
  sync-role:
    enabled: true    # only flag remains
```

The shared `keycloak` block is used by both `sync-role` and `group-role-authorization`.

### UserInfoAuthenticationConverter returns Mono

`UserInfoAuthenticationConverter` now implements `Converter<OidcIdToken, Mono<UserinfoAuthentication>>` (was
`Converter<OidcIdToken, UserinfoAuthentication>`). Custom converters must return `Mono`.

## What's New

### Group-Role Authorization (new)

Alternative authorization strategy using group-based role resolution with layered caching.

**Strategy is exclusive** — choose one via config:

- `group-role-authorization.enabled: false` (default) → reads `resource_access` claim from JWT
- `group-role-authorization.enabled: true` → reads `role_groups` claim, resolves via L1/L2/L3 cache

```yaml
f8a:
  security:
    apisix:
      resource-server:
        keycloak:
          server-url: https://kc.example.com
          realm: my-realm
          client-id: admin-cli
          client-secret: secret
        group-role-authorization:
          enabled: true
          claim-name: role_groups
          l1:
            ttl: 60s                           # Caffeine in-process cache
          l2: # omit entire block = L1-only mode
            ttl: 5m                            # Redis SET TTL
            invalidation-channel: "group-role-changes"
            key-prefix: "auth-group-role:"
            redis:
              host: localhost
              port: 6379
              password: ""
              database: 0
```

**Key classes:**

| Class                                | Purpose                                                                          |
|--------------------------------------|----------------------------------------------------------------------------------|
| `GroupRoleResolver`                  | L1 Caffeine → L2 Redis (optional) → L3 fetcher cache cascade                     |
| `GroupRoleAuthenticationConverter`   | `Converter<OidcIdToken, Mono<UserinfoAuthentication>>` using group-role resolver |
| `GroupRoleFetcher`                   | `@FunctionalInterface` for L3 source (default: Keycloak `group-by-path` API)     |
| `GroupRoleInvalidator`               | Publishes Redis pub/sub invalidation messages                                    |
| `ReactiveGroupRoleAutoConfiguration` | Wires everything when enabled                                                    |

**Default L3 fetcher:** calls `GET /admin/realms/{realm}/group-by-path/{groupName}`, extracts `clientRoles` map →
flattens to `clientId:roleName` format. Override by providing your own `GroupRoleFetcher` bean.

**Cache invalidation:** `PUBLISH group-role-changes "groupName"` — or inject `GroupRoleInvalidator`:

```java
groupRoleInvalidator.invalidate("admins").

subscribe();
```

See `docs/AUTH_WITH_CACHE.md` for full architecture and design decisions.

### Customizable authentication converter

`ApisixAuthenticationManager` now accepts a pluggable converter via constructor injection. The default
`UserInfoAuthenticationConverter` is registered with `@ConditionalOnMissingBean` — provide your own bean to override.

Authentication flow mirrors Spring Security's `JwtReactiveAuthenticationManager`:

```java
.flatMap(this::decode)
.

flatMap(this.userInfoAuthenticationConverter::convert)
```

### Keycloak API expansion

**Group-by-path:**

```java
keycloak.realm().

groupByPath("my-org/team-a");  // Mono<GroupRepresentation>
```

**Group role-mappings:**

```java
var group = keycloak.groups().get(groupId);
group.

roles().

realmLevel().

listAll();                    // Flux<RoleRepresentation>
group.

roles().

clientLevel(clientUUID).

listAll();          // Flux<RoleRepresentation>
group.

roles().

clientLevel(clientUUID).

add(rolesToAdd);    // Mono<Void>
group.

roles().

clientLevel(clientUUID).

remove(rolesToRemove); // Mono<Void>
```

**Client scopes (realm-level):**

```java
keycloak.realm().

clientScopes().

findAll();                // Flux<ClientScopeRepresentation>
keycloak.

realm().

clientScopes().

create(scopeRep);         // Mono<String>
keycloak.

realm().

clientScopes().

get(id).

toRepresentation(); // Mono<ClientScopeRepresentation>
keycloak.

realm().

clientScopes().

get(id).

protocolMappers().

getMappers(); // Flux<ProtocolMapperRepresentation>
```

**Client default/optional scopes:**

```java
var client = keycloak.clients().get(clientUuid);
client.

getDefaultClientScopes();                          // Flux<ClientScopeRepresentation>
client.

addDefaultClientScope(scopeId);                    // Mono<Void>
client.

getOptionalClientScopes();                         // Flux<ClientScopeRepresentation>
client.

addOptionalClientScope(scopeId);                   // Mono<Void>
```
