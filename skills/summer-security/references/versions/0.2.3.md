# Summer 0.2.3 — summer-security

Released: 2026-03-09

## Features

- **`KeycloakException` mapping expansion** — from ~5 to ~50 cases. Covers auth, user state, token
  lifecycle, session, client credentials, OAuth parameters, PKCE, identity provider errors. Each
  case maps to a typed error code with an appropriate HTTP status. Unknown errors fall back to
  `keycloak.<errorDescription>` (500). See [keycloak-error-map.md](../keycloak-error-map.md) for
  the full table.
- **`KeycloakRoleSynchronizer` enabled by default.** Auto-config now uses `matchIfMissing = true`
  on `sync-role.enabled`, so role sync activates whenever `sync-role.server-url` is configured —
  no explicit `enabled: true` needed. Disable with `sync-role.enabled: false`.
- **`RoleDefinitionScanner` registered as a Spring bean** by the auto-configuration. Inject it
  even when role sync is not active.

## Breaking changes

- **Property rename:** `f8a.security.apisix.resource-server.sync-role.enable` →
  `f8a.security.apisix.resource-server.sync-role.enabled`.

## Bug fixes

- Improved code readability and error logging in `ApisixAuthenticationManager`.

## Migration from 0.2.2

1. Search-replace `sync-role.enable` → `sync-role.enabled` in every `application*.yml` and
   per-environment override.
2. Drop explicit `enabled: true` if `sync-role.server-url` is set — it's the new default.
3. If you depend on a specific `KeycloakException` error code, re-test against the expanded
   mapping table (most legacy 500s now have typed codes).
