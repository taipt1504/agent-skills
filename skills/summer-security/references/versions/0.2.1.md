# Summer 0.2.1 — summer-security

Released: 2026-02-26

## Module renames (BREAKING)

| Before | After | Notes |
|---|---|---|
| `summer-security-core` (artifact) | `summer-jwt-resource-server` | Update `build.gradle` dependency |
| `io.f8a.summer.security.common.*` | `io.f8a.summer.security.common.*` (in `summer-jwt-resource-server`) | Package unchanged, module changed |
| `io.f8a.summer.security.jwt.*` | `io.f8a.summer.security.jwt.*` (in `summer-jwt-resource-server`) | Package unchanged, module changed |
| (none) | `summer-apikey-resource-server` (new module) | Add dependency if API-key auth used |

`ReactiveApiKeyAuthentication` and `ReactiveApiKeyWebFilter` moved from the old `security-core`
into the new `summer-apikey-resource-server`.

## Removed

- `ReactiveSecurityProperties` (unused config) — deleted from `security-autoconfigure`.
- `ApiKeyAuthentication` — replaced by `ReactiveApiKeyAuthentication`.
- `UserinfoServerAuthenticationEntryPoint` — replaced by `SecurityErrorResponseWriter`.
- `DefaultAuthenticationException` and `DefaultJwtConverter` moved from `security-autoconfigure` to
  `jwt-resource-server` and rewritten.

## Behavioural changes

- `ReactiveApisixCustomizer` constructor now takes `ObjectMapper` (was no-arg).
- `UserinfoGrantedAuthoritiesConverter.authoritiesClaimName` is now a protected method (was a field).

## Features

- New `SecurityErrorResponseWriter` utility in `summer-jwt-resource-server` — writes
  `JsonErrorResponse` JSON for security filter errors using `ObjectMapper` and MDC-based `traceId`.
- New `DefaultAuthenticationException` and `DefaultJwtConverter` in `summer-jwt-resource-server`
  with ObjectMapper-based JWT claim mapping.

## Migration from 0.2.0

1. Update Gradle:
   ```diff
   - implementation 'io.f8a.summer:summer-security-core'
   + implementation 'io.f8a.summer:summer-jwt-resource-server'
   ```
   Add `summer-apikey-resource-server` only if the service uses API-key auth.
2. Pass `ObjectMapper` to any direct `ReactiveApisixCustomizer` instantiation in tests/custom config.
3. If you subclassed `UserinfoGrantedAuthoritiesConverter`, override `authoritiesClaimName()` instead of setting the field.
