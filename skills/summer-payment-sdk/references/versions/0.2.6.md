# Summer 0.2.6 — summer-payment-sdk

Released: 2026-04-02

UFID display format switches hex → Crockford Base32 across all payment-sdk surfaces; annotations are renamed accordingly.

## Breaking changes

### `@TXN` → `@TX`

```java
// Before
@TXN Ufid transferId;
// After
@TX Ufid transferId;
```

Wire format changes from `TXN_<32-char-hex>` to `TX_<26-char-Crockford-Base32>`.

### `@Hex` → `@Compact`

```java
// Before
@Hex Ufid id;
// After
@Compact Ufid id;
```

Wire format changes from 32-char hex to 26-char Crockford Base32 (lowercase-tolerant, `0/O/1/I/L` aliased).

### `LedgerOperation.CAPTURE` → `LedgerOperation.POST`

The enum constant renamed for terminology consistency with the rest of the ledger model (POST/VOID/REVERSE).

## Features

### WebFlux `String → Ufid` converter auto-registered

Path variables and query params bind to `Ufid` directly without service-side converters.

## Migration from 0.2.5

1. Search-replace `@TXN` → `@TX`.
2. Search-replace `@Hex` → `@Compact`.
3. Search-replace `LedgerOperation.CAPTURE` → `LedgerOperation.POST`.
4. Re-publish any persisted strings that embedded the old hex form — the encoding changed; consumers reading old strings will not parse them as the new format.

## Verification

- `@TX`-annotated fields serialize as `"TX_..."` 28-char strings (26 base32 chars + `TX_` prefix).
- `@Compact`-annotated fields serialize as 26-char Crockford Base32 without prefix.
- `LedgerOperation.POST` round-trips through Jackson; `CAPTURE` references fail at compile time.
