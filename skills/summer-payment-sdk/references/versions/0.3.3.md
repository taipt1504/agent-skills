# Summer 0.3.3 — summer-payment-sdk

Released: 2026-05-06

`@Link` removed in favour of `@Compact` (BREAKING). New producer-routing vocabulary shared between Payment Service and Core-Ledger.

## Breaking changes

### `@Link` annotation removed

```java
// Before (0.3.1 → 0.3.2)
@Link Ufid relatedTransferId;

// After (0.3.3+)
@Compact Ufid relatedTransferId;
```

The `LINK_` prefix discriminator is no longer carried as a separate annotation. Replace `@Link` with `@Compact` on every field, and update any string-side parsing that expected an incoming `LINK_<...>` prefix — values now arrive as bare Crockford Base32 (no prefix).

## Features

### Producer-routing vocabulary (`io.f8a.summer.payment.domain.producer.*`)

Six new shared types — both Payment Service and Core-Ledger materialize the same YAML rows byte-identically against this contract.

| Type | Role |
|---|---|
| `Direction` (enum) | `TOPUP, WITHDRAW, TRANSFER, REFUND, CREDIT, DEBIT, FEE`. Each carries its TigerBeetle transfer code, aligned with the legacy `TransactionType` codes for cross-flow report consistency. |
| `ProducerId` (record) | `parse(String)` enforces `<channel>.<partner>.<direction>.v<N>` (external) or `internal.<flow>.<variant>.v<N>` (internal); exposes `channel()`, `partner()`, `version()`, `isInternal()`. |
| `ProducerStatus` (enum) | `ACTIVE, DEPRECATED, SUNSET, ORPHANED, EMERGENCY_DISABLED`. |
| `ProducerSpec` (record) | Jackson-deserialized canonical YAML row. `@JsonIgnoreProperties(ignoreUnknown = true)` lets each service ignore blocks it doesn't consume. |
| `YamlHash` | SHA-256 over YAML bytes; both services hash byte-identically. Use to verify both sides loaded the same revision. |
| `ProducerEventFields` (constants) | `PRODUCER_ID`, `FEE_PRODUCER_ID` — wire-side field names so producer and consumer never disagree on wire format. |

### Platform BOM pins payment-sdk by explicit coordinates

`summer-platform` now pins `summer-payment-sdk` via `io.f8a.summer:summer-payment-sdk:${paymentSdkVersion}` (where `paymentSdkVersion` is a top-level Gradle property in upstream). The BOM advertises a known-good version per Summer release while still letting consumers override to a newer payment-sdk independently.

## Migration from 0.3.1 / 0.3.2

1. Search-replace `@Link` → `@Compact` on every field; remove any string-side `"LINK_"` prefix handling.
2. (Optional) adopt the producer-routing vocabulary when wiring services to a shared `producers.yaml`:
   - Parse the YAML into `ProducerSpec`.
   - Verify both services produce the same `YamlHash` at boot.
   - Use `ProducerId.parse(...)` instead of hand-split string parsing.

## Verification

- A round-trip of a record that previously used `@Link` now serializes as bare Crockford Base32 — no `LINK_` prefix in the JSON.
- `ProducerId.parse("zalo.zlp.topup.v1").channel()` returns `"zalo"`; `parse("internal.refund.v1").isInternal()` returns `true`.
- Payment Service and Core-Ledger emit the same `YamlHash.sha256(yamlBytes)` for the same input file.
