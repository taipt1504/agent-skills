# Summer 0.2.5 — summer-payment-sdk (module introduction)

Released: 2026-03-30

`summer-payment-sdk` ships in 0.2.5 alongside the introduction of the `Ufid` type in `summer-core`. This is the introduction overlay.

## Features

### Module added

`io.f8a.summer:summer-payment-sdk` — single shared jar for payment event DTOs, payment domain enums, and Ufid Jackson annotations. Replaces ad-hoc per-service copies.

### Prefix annotations (pre-rename)

Pre-0.2.6 names — rename to current forms in 0.2.6 (see that overlay).

- `@JE` — `JE_<hex>` prefix (journal entries).
- `@TXN` — `TXN_<hex>` prefix (transfers). **Renamed to `@TX` in 0.2.6.**
- `@SE` — `SE_<hex>` prefix (saga executions).
- `@UfidPrefix("XX")` — custom prefix, also usable as a meta-annotation to define new aliases.
- `@Hex` — 32-char hex form, no prefix. **Renamed to `@Compact` in 0.2.6 (and the encoding changed hex → Crockford Base32 in the same release).**

### Payment domain enums

`io.f8a.summer.payment.domain.*` — `LedgerAccountCodes`, `LedgerAccountType`, `LedgerEntryDirection`, `LedgerOperation` (with `CAPTURE` enum — renamed `POST` in 0.2.6), `TransactionChannel`, `TransactionStatus`, `TransactionType`.

### `UfidConverter` (R2DBC)

R2DBC converter for `Ufid ↔ UUID` storage. Auto-registered by `summer-data-r2dbc`.

## Breaking changes

None (module introduction).

## Migration from 0.2.4

Optional adoption — to share payment DTOs:

1. Add the dependency: `implementation 'io.f8a.summer:summer-payment-sdk'`.
2. Delete service-local enum copies of `LedgerOperation`, `TransactionType`, etc., and import from `io.f8a.summer.payment.domain.*`.
3. Migrate event DTOs to the shared types as they become available.
